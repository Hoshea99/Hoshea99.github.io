docker使用

发表于 2026-04-20 分类于 linux

docker能够让我们在开发机器上跨系统编译，有效地提高了工作效率。这篇文章将重点分析docker的核心逻辑和基本操作。

核心原理

Docker不像虚拟机一样，模拟整个硬件和内核。他共享了宿主机的Linux内核，在用户态构建了一套独立的文件系统。

所以只要内核是兼容的，就能在Ubuntu22.04的机器上 运行一个Ubuntu20.04甚至是centos的环境，而不污染宿主机（windows不行，因为内核不一样）

  • docker三要素
    • Image（镜像）：静态的“只读图纸”，包含了系统底座、依赖库和编译工具。它是分层存储的，可以被多次复用。
    • Container (容器)： 镜像运行后的“动态实例”。每次 docker run 都是基于镜像盖的一间新房子。
    • Mount (挂载)： 连接宿主机与容器的“传送门”。通过 -v 参数，将宿主机的代码目录映射到容器内部，实现“代码在宿主机，环境在容器，产物在宿主机”的高效流程。

常用操作

管理镜像

命令 说明
docker images 查看本地已下载的所有镜像
docker pull <image_name> 从远程仓库（Docker Hub）拉取镜像
docker rmi <image_id> 删除不再需要的镜像

容器操作

命令 说明
docker run -it -v <宿主机路径>:<容器路径> <镜像名> /bin/bash 最常用： 创建并进入一个挂载了本地代码的新环境
docker ps 查看当前正在运行的容器
docker ps -a 查看所有容器（包括已停止的）
docker stop <ID> 停止一个正在运行的容器
docker start -i <ID> 重启并进入一个之前停止的旧容器
docker rm <ID> 删除已经不用的容器实例

  • 第一次使用docker run，后面如果要先退出来就用docker stop，然后docker start可以继续在原环境中进行工作。在docke中想出来可以通过ctrl + D出去。

  • docker run -it -v <宿主机路径>:<容器路径> <镜像名> /bin/bash

    • -v: 这是最关键的 Volume（卷）挂载。它在宿主机和容器之间修了一条“隧道”。你在宿主机的这个目录下改代码，容器里立刻同步，反之亦然
    • 可以再加一个参数：--user $(id -u):$(id -g) 这条命令把宿主机的用户 ID（UID）和组 ID（GID）传给容器。这样容器里的操作就像是你本人在操作，生成的代码和编译产物权限完全正常。否则，在docker生成的成果物是root的
