linux网卡网口相关

发表于 2026-04-11 分类于 linux

量化工作中需要通过网口接收行情，所以我们对网口/网卡等知识要有一个基础的了解。

测试接收数据

即监听所有网卡的5555端口数据。-X显示十六进制或者ASCII两种格式的包体内容，-nn指的是不进行域名和端口名称的解析。

但是不能开启混杂模式，如果要开启混杂模式，不能监听any，只能监听特定网卡。

当你使用 any 时，由于架构限制，内核不允许通过这个虚拟入口开启底层硬件的混杂模式。这会导致以下差异：

场景 -i eth0 (混杂模式) -i any
发给本机的包 ✅ 能抓到 ✅ 能抓到
本机发出的包 ✅ 能抓到 ✅ 能抓到
广播/组播包 ✅ 能抓到 ✅ 能抓到
发给同交换机下别人的包 ✅ 能抓到 (只要电信号路过这根线) ❌ 抓不到

查看本机网卡信息

可以看本机网卡信息。

  • lo是本地回环

  • NO-CARRIER指的是没接线也不用管

  • PROMISC混杂模式，一般用来接收行情

  • DOWN目前是关掉的

    • 为什么有时候网卡没有分配ipv4的地址呢，他其实是只听不讲的，只需要对组播MAC地址过滤并接收数据就好，不配置IPV4地址可以避免网卡被外部扫描或者攻击。
