enable_if和void_t

发表于 2025-12-20 分类于 C++

C++模版中有一个很重要的特性：SFINAE，即替换失败并非错误。在函数头或者模版参数列表（并非函数体中）发生了语法无效的问题，编译期不会停止，而是会将这个模版从候选中移除。

在模板编程中，我们经常遇到一个尴尬：我想写 T().name 来探测类型，但 T 可能没有默认构造函数（比如它只有带参数的构造函数 T(int x)）。这时候直接写 T() 编译器会报错。

std::declval<T>() 的作用就是： 告诉编译器“假设我这里有一个 T 类型的对象”。

  • 它不需要构造函数。
  • 它不会真的创建对象（不会占用内存，也不会运行）。
  • 它只能用在“不求值语义”（Unevaluated Context）中，比如 sizeof 或 decltype。

我们可以发现 void_t更适合去判断是否存在某个成员

最后用gemini的一个比喻进行总结：

std::enable_if 是一个 门卫：他手里有一张名单（布尔值），名单上有你你就进，没有你就不让进。

std::void_t 是一个 体检医生：他会让你做一套动作（表达式）。如果你能做出来，他就给你盖个“通过”的章（void）；如果你做不出来，他就摇摇头。
